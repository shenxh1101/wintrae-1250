## 1. 产品概述

剧场巡演统筹管理系统是一款面向小型剧场巡演团队和场馆对接人的一站式演出落地管理平台。它将分散在微信、邮件、Excel 中的演出信息统一整合到 5 个核心页面中，帮助统筹人员和场馆人员高效协作，确保每场演出顺利落地。

- 核心目标：将零散沟通集中到少量页面，降低沟通成本，减少信息遗漏
- 目标用户：巡演统筹、场馆对接人、演出技术负责人
- 产品价值：提升巡演落地效率，标准化交接流程，实现场次全生命周期管理

## 2. 核心功能

### 2.1 用户角色

| 角色 | 登录方式 | 核心权限 |
|------|----------|----------|
| 巡演统筹 | 账号登录 | 全部功能，创建/编辑场次，导出交接摘要 |
| 场馆对接人 | 账号登录 | 查看指定场次，更新物料状态，上传场地照片，记录沟通 |
| 技术负责人 | 账号登录 | 查看设备需求，更新技术准备状态 |

### 2.2 功能模块

1. **日历总览页**：按城市排期展示，快速查看全巡演场次分布与状态
2. **场次详情页**：单场演出全量信息，进退场时间、人员名单、设备需求、待确认问题
3. **物料清单页**：海报、票夹等物料状态跟踪，标记制作/寄送/到位进度
4. **沟通记录页**：集中记录所有对接沟通，支持按场次、按主题筛选
5. **结算归档页**：票务与分成口径记录，单场交接摘要生成，历史归档查询

## 3. 核心流程

### 3.1 巡演筹备流程

统筹创建新场次 → 复制上场配置 → 填写基本信息 → 与场馆确认进退场时间 → 登记设备需求 → 制作物料清单 → 双方沟通确认 → 演出执行 → 结算归档

```mermaid

    B --> C["复制上场配置"]

    D --> E["场馆更新物料状态"]
    E --> F["上传场地照片"]
    F --> G["沟通记录同步"]
    G --> H["演出当天执行"]
    H --> I["生成交接摘要"]
    I --> J["结算归档"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深酒红色 #722F37，体现剧场艺术感与专业度
- **辅助色**：暖金色 #D4A574，作为强调色和状态高亮
- **中性色**：炭灰 #1A1A2E、米白 #F5F0E8，营造高雅沉稳的氛围
- **按钮风格**：圆角 6px，主按钮为酒红底色配金色文字边框
- **字体**：标题使用 'Noto Serif SC' 衬线字体，正文使用 'Inter' 无衬线字体
- **布局风格**：左侧导航栏 + 右侧主内容区，卡片式信息模块
- **图标风格**：线性图标，酒红色主色，20px 标准尺寸
- **整体调性**：精致、专业、戏剧感，像剧场节目单一样的视觉体验

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 日历总览 | 月份日历 | 大网格日历，场次卡片嵌入日期格，状态色标 |
| 日历总览 | 侧边统计 | 城市分布统计卡，状态概览数字 |
| 场次详情 | 信息分区 | 折叠式卡片分区，金色分割线，戏剧感排版 |
| 场次详情 | 人员名单 | 头像 + 姓名 + 角色标签的列表布局 |
| 物料清单 | 状态流 | 横向四步状态条，当前状态高亮 |
| 沟通记录 | 时间线 | 左侧时间轴，右侧气泡式沟通内容 |
| 结算归档 | 摘要卡 | 仿节目单样式的交接摘要卡片 |

### 4.3 响应式

- 桌面端优先设计（1280px 起）
- 平板端（768px）：导航折叠为抽屉，内容区单列
- 手机端：底部 Tab 导航，卡片堆叠布局
- 触控优化：最小点击区域 44x44px

### 4.4 动效与交互

- 页面切换：淡入 + 轻微上移，时长 300ms
- 卡片悬停：阴影加深 + 轻微上浮，时长 200ms
- 状态切换：金色光晕闪烁提示
- 时间线加载：自上而下依次出现的渐入效果
